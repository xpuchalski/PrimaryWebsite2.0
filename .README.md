# Adding a New Page (Auto Navbar)

Your navbar is generated automatically from page files in `src/pages`.

## How It Works

The app scans files matching `*Page.jsx` inside `src/pages` and reads exported `page` metadata.

Auto-registration is implemented in:
- `src/pages/registry.js`

Navbar rendering is implemented in:
- `src/components/SiteLayout.jsx`

## Add A New Page

1. Create a new file in `src/pages` with a name ending in `Page.jsx`.

Example filename:
- `AboutMePage.jsx`

2. Export a `page` object and a default React component.

Use this template:

```jsx
export const page = {
  path: "/about-me",
  title: "about me",
  navLabel: "about me", // optional, defaults to title
  order: 4,              // optional, lower number = earlier in nav
  showInNav: true        // optional, set false to hide from nav
};

function AboutMePage() {
  return (
    <section className="panel prose">
      <h1>About Me</h1>
      <p>Your content here.</p>
    </section>
  );
}

export default AboutMePage;
```

3. Save the file.

That is it. The route and navbar entry will be created automatically.

## Notes

- `path` is the URL for the page.
- `title` is used for sorting and fallback label.
- `navLabel` controls exact text shown in navbar.
- `order` controls nav order.
- If two pages have the same `order`, they are sorted alphabetically by `title`.
- Set `showInNav: false` for hidden routes.

## Quick Check

Run:

```bash
npm run dev
```

Then open your new route, for example:
- `http://localhost:5173/about-me`
